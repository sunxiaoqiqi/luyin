# 简录制器产品需求文档

## 1. 产品定位

简录制器是一款面向教程、演示、课程、产品讲解场景的桌面录制产品。产品核心目标是让用户可以在一个应用内完成「录制屏幕或指定窗口 -> 自动保存素材 -> 简单整理多段素材 -> 直接生成剪映草稿」的闭环。

本产品参考 `jianying-draft-tools` skill 中已有能力进行设计：

- 素材扫描：识别指定文件夹中的视频、图片素材。
- 素材元数据：识别与视频同名的 `_events.json`，用于后续智能缩放、点击位置等录制事件。
- 素材编排：通过配置文件把素材组织为顺序片段，并生成剪映草稿。
- 封面提取：使用 ffmpeg 提取视频首帧，生成素材缩略图。

当前产品不直接做完整专业剪辑软件，而是优先解决录制后的第一步整理和轻编排。

## 2. 目标用户

- 老师、培训师：录制板书、课件讲解、知识点拆分。
- 产品经理、设计师、开发者：录制功能演示、问题复现、操作说明。
- 内容创作者：快速录制多个讲解片段，再进入剪映或其他工具精剪。

## 3. 核心问题

用户现在需要分别使用录屏软件、文件管理器和剪辑软件，流程割裂：

- 录制前需要反复确认保存位置。
- 一次录制失败后难以快速补录其中一段。
- 多段素材录完后需要手动找文件、改名、排序。
- 轻微调整顺序、删除废片、合并片段的成本过高。
- 如果要进入剪映，还需要额外导入素材并重建时间线。

## 4. 产品目标

MVP 版本必须完成以下目标：

- 用户可以直接开始录制。
- 用户可以设置录制保存地址。
- 支持普通录制和多段录制两种模式。
- 普通录制结束后，文件自动出现在指定位置。
- 多段录制结束后，片段自动进入录制面板的素材列表。
- 用户可以对多段素材进行初步编排：预览、重命名、删除、排序、裁剪首尾、合并导出。
- 支持直接生成剪映草稿；项目配置 JSON 作为内部中间产物保留。

## 5. 产品范围

### 5.1 MVP 范围

- 录制源选择：整个屏幕、指定窗口。
- 录制范围设置：支持默认画幅比例和自定义范围。
- 音频选择：系统声音、麦克风、系统声音 + 麦克风、静音。
- 保存地址设置：选择文件夹、展示剩余空间、记住上次路径。
- 录制模式：
  - 普通录制：一次录制生成一个视频文件。
  - 多段录制：每次停止生成一个片段，并加入当前项目素材池。
- 录制控制：开始、暂停、继续、停止、取消。
- 多段素材面板：
  - 素材卡片展示缩略图、文件名、时长、创建时间、大小。
  - 支持拖拽排序。
  - 支持重命名、删除、打开文件位置。
  - 支持片段预览。
  - 支持设置入点、出点。
- 初步编排：
  - 按素材面板顺序合并为一个视频。
  - 生成剪映草稿。
  - 保留项目配置 JSON，便于排错、复用和二次生成。
- 基础异常处理：
  - 保存路径不可写。
  - 磁盘空间不足。
  - 录制权限不足。
  - ffmpeg 不可用。
  - 录制中应用异常退出后的临时文件恢复。
  - 关闭应用时同步清理主进程、渲染进程、录制任务和所有子进程。

### 5.2 暂不包含

- 多轨专业剪辑。
- 复杂字幕编辑。
- 多段编排中的字幕能力。
- 摄像头画中画。
- 高级转场、滤镜、贴纸、调色。
- 云端同步和团队协作。
- 账号体系。

## 6. 用户流程

### 6.1 首次使用流程

1. 用户打开应用。
2. 应用提示选择默认保存地址。
3. 用户选择录制源和音频源。
4. 用户选择普通录制或多段录制。
5. 用户点击开始录制。

### 6.2 普通录制流程

1. 用户选择「普通录制」。
2. 点击「开始录制」。
3. 录制中可以暂停、继续、停止。
4. 停止后生成一个视频文件。
5. 文件保存到指定目录。
6. 应用显示录制完成结果：
   - 文件名
   - 时长
   - 文件大小
   - 保存路径
   - 打开文件位置
   - 重新录制

### 6.3 多段录制流程

1. 用户选择「多段录制」。
2. 点击「开始录制」录第一段。
3. 停止后，该片段自动进入素材面板。
4. 用户可以继续点击「录制下一段」。
5. 每段录制完成后都追加到素材面板底部。
6. 用户在素材面板中预览、删除、重命名、排序、裁剪。
7. 用户点击「合并导出」或「生成剪映草稿」。

## 7. 页面结构

### 7.1 主录制页

主录制页由三块组成：

- 顶部工具栏：录制模式、保存地址、录制源、音频源、画质设置。
- 中间预览区：显示当前录制源预览、录制状态、计时器。
- 底部控制栏：开始、暂停、继续、停止、取消。

### 7.2 多段素材面板

多段模式下，右侧或底部展示素材面板：

- 当前项目名称。
- 片段数量、总时长、预计导出大小。
- 素材列表。
- 编排操作区：合并导出、生成剪映草稿、打开草稿目录。

### 7.3 设置页

- 默认保存地址。
- 默认录制格式：mp4。
- 默认录制范围：16:9、4:3、3:4、9:16、1:1、自定义。
- 分辨率：原始、1080p、720p。
- 帧率：30 fps、60 fps。
- 码率：自动、低、中、高。
- 文件命名规则。
- 是否记录鼠标点击事件。
- 是否记录鼠标移动轨迹。
- 是否生成 `_events.json`。
- ffmpeg 路径检测。

## 8. 功能需求

### 8.1 保存地址设置

用户故事：作为用户，我希望设置录制文件保存位置，这样录完后不需要再手动寻找或移动文件。

需求：

- 支持选择本地文件夹。
- 路径需持久化保存。
- 显示当前路径。
- 显示路径是否可写。
- 显示磁盘剩余空间。
- 路径不可用时阻止开始录制，并提示原因。

验收标准：

- 关闭应用后重新打开，仍保留上次保存地址。
- 保存地址不存在时，应用提示重新选择。
- 无写入权限时，开始录制按钮不可用。

### 8.2 普通录制

用户故事：作为用户，我希望像普通录屏软件一样直接录制，停止后得到一个视频文件。

需求：

- 点击开始后进入录制中状态。
- 录制中显示计时器。
- 支持暂停、继续、停止。
- 停止后自动保存为 mp4。
- 文件命名默认格式：`recording_YYYYMMDD_HHmmss.mp4`。
- 如果开启事件记录，同时生成 `recording_YYYYMMDD_HHmmss_events.json`。

验收标准：

- 录制 10 秒后停止，指定目录出现可播放 mp4。
- 普通录制不强制创建项目。
- 录制完成后可打开文件所在位置。

### 8.2.1 录制范围设置

用户故事：作为用户，我希望快速选择常用平台画幅，也可以手动设置录制区域，这样录制出来的内容可以直接适配发布场景。

需求：

- 支持以下录制范围预设：
  - `16:9`：YouTube、横屏课程、通用桌面演示。
  - `4:3`：经典课件、传统演示比例。
  - `3:4`：小红书、竖向内容封面或笔记视频。
  - `9:16`：抖音、短视频竖屏内容。
  - `1:1`：正方形内容。
  - `Custom`：自定义宽高比或手动框选区域。
- 支持设置默认录制范围。
- 选择预设后，录制预览区显示对应裁切框。
- 用户可以拖动裁切框位置。
- `Custom` 模式下支持输入宽度、高度、X、Y，或在预览区拖拽调整。
- 录制范围不得超出当前录制源边界。
- 默认建议为 `16:9`。

验收标准：

- 用户可在主录制页切换 16:9、4:3、3:4、9:16、1:1、Custom。
- 设置默认范围后，重新打开应用仍保留该默认值。
- 使用 9:16 录制时，输出视频为竖屏比例。
- Custom 范围超出屏幕边界时，应用自动修正或提示。

### 8.3 多段录制

用户故事：作为用户，我希望把一次内容拆成多段录制，便于失败时只重录某一段。

需求：

- 多段模式下，每次停止生成独立视频片段。
- 每个片段自动加入当前录制项目。
- 素材面板实时刷新。
- 片段默认命名：`segment_001.mp4`、`segment_002.mp4`。
- 支持继续录制下一段。
- 支持删除某段，删除时二次确认。
- 支持重命名片段。
- 支持拖拽排序。
- 支持标记片段状态：待处理、已确认、废弃。

验收标准：

- 连续录制 3 段后，素材面板显示 3 个片段。
- 调整排序后，合并导出按新顺序生成视频。
- 删除片段后，不再参与合并和草稿生成。

### 8.4 素材预览与基础编排

用户故事：作为用户，我希望在录制面板上快速看每一段，并做最基础的整理。

需求：

- 点击素材卡片可预览。
- 显示缩略图，优先用 ffmpeg 提取首帧。
- 支持设置入点和出点。
- 支持恢复原始裁剪。
- 支持片段备注。
- 支持一键按创建时间排序。
- 支持导出素材清单 JSON。

验收标准：

- 每个视频片段都有缩略图或默认图标。
- 设置入点/出点后，合并导出只使用裁剪后的范围。
- 素材清单中包含路径、时长、裁剪信息、排序信息。

### 8.5 合并导出

用户故事：作为用户，我希望把多段录制快速合并为一个视频，用于发布或继续剪辑。

需求：

- 按当前素材排序合并。
- 支持导出文件名设置。
- 默认导出到当前项目目录。
- 支持无转码快速合并；如格式不一致则自动转码。
- 导出完成后显示结果。

验收标准：

- 3 个片段可以合并成一个 mp4。
- 合并后总时长等于各片段裁剪后时长之和，允许 1 秒内误差。
- 导出失败时显示具体错误。

### 8.6 剪映草稿衔接

用户故事：作为用户，我希望多段素材可以快速进入剪映继续精剪。

需求：

- 直接生成剪映草稿。
- 生成与 `material_editor.py` 兼容的项目配置 JSON，作为内部中间产物保存到项目目录。
- 配置中包含：
  - `project_name`
  - `resolution`
  - `fps`
  - `materials_folder`
  - `associations`
  - 每段素材的 `file`、`type`、`events_json`、`layout`
- 点击「生成剪映草稿」后，调用现有素材编排脚本生成草稿。
- 如果生成失败，保留配置文件和错误日志。

验收标准：

- 多段录制项目可以生成剪映草稿。
- 生成草稿时同步保留配置 JSON。
- 配置 JSON 可被素材编排脚本读取。
- 带 `_events.json` 的片段在配置中正确关联。

### 8.7 录制事件记录

用户故事：作为用户，我希望录制时记录鼠标点击、焦点位置等信息，后续可自动做智能缩放。

说明：鼠标轨迹指完整记录鼠标在录制过程中的移动路径；点击事件只记录点击发生的时间和位置。MVP 默认只记录点击事件，鼠标轨迹作为可选增强项，不影响录制主流程。

需求：

- 可开关事件记录。
- 记录鼠标点击时间、屏幕坐标、窗口坐标。
- 鼠标移动轨迹默认不记录，可在后续版本提供开关。
- 记录格式保存为同名 `_events.json`。
- 普通录制和多段录制都支持。
- 多段录制中，每段生成自己的 events 文件。

建议事件结构：

```json
{
  "version": "1.0",
  "video_file": "segment_001.mp4",
  "screen": {
    "width": 1920,
    "height": 1080
  },
  "events": [
    {
      "type": "click",
      "time": 1.235,
      "x": 960,
      "y": 540,
      "button": "left"
    }
  ]
}
```

验收标准：

- 开启事件记录后，每个视频旁边都有对应 `_events.json`。
- 素材扫描能够识别 `has_events = true`。

### 8.8 应用关闭与后台进程清理

用户故事：作为用户，我希望点击关闭按钮后软件彻底退出，不在任务管理器中继续残留进程。

需求：

- Windows 下点击主窗口关闭按钮时，默认行为为退出应用，而不是最小化到托盘或隐藏窗口。
- 如果未来增加托盘常驻能力，必须提供明确设置项，默认关闭托盘常驻。
- 应用退出时需要停止正在进行的录制任务。
- 应用退出时需要关闭或终止所有由应用启动的子进程，包括 ffmpeg、Python 草稿生成脚本、缩略图提取任务等。
- 应用退出时需要释放录屏流、麦克风流、系统音频流。
- 应用退出前如存在正在写入的视频文件，需要优先完成封装；无法完成时保留临时文件并记录恢复信息。
- 应用退出时需要清理 IPC 监听、定时器和后台队列。

验收标准：

- 安装版应用启动后，点击窗口关闭按钮，任务管理器中不再保留本应用相关进程。
- 录制中点击关闭，应用提示是否停止录制并退出；确认后不残留录制进程。
- 正在生成剪映草稿时点击关闭，应用提示任务正在运行；确认退出后终止草稿生成子进程。
- 连续打开和关闭应用 5 次，不出现进程数量累积。

## 10. 技术方案建议

### 10.1 桌面端

建议使用 Electron 作为桌面容器，原因：

- skill 中 `material-editor` 已经使用 Electron。
- Electron 支持窗口、文件系统、对话框、IPC。
- 后续可接入 `desktopCapturer` 选择屏幕或窗口。

### 10.2 录制实现

可选方案：

- Electron + Chromium MediaRecorder：实现快，适合 MVP。
- ffmpeg 录制：控制力更强，适合稳定生产版。

MVP 建议优先使用 MediaRecorder 完成录制，再用 ffmpeg 做格式整理、缩略图、合并导出。

### 10.3 后端能力

主进程提供 IPC：

- `select-folder`
- `get-save-dir`
- `set-save-dir`
- `start-recording`
- `set-capture-area`
- `pause-recording`
- `resume-recording`
- `stop-recording`
- `cancel-recording`
- `scan-materials`
- `get-video-thumbnail`
- `merge-segments`
- `generate-jianying-config`
- `generate-jianying-draft`
- `cleanup-child-processes`

### 10.4 复用 skill 能力

- 复用 `material_scanner.py` 的素材扫描逻辑。
- 复用 `material_editor.py` 的剪映草稿生成逻辑。
- 复用 `_events.json` 命名规则，保证智能缩放兼容。
- 复用 ffmpeg 首帧提取作为素材缩略图能力。

## 11. 非功能需求

- 稳定性：录制中崩溃后，尽量恢复临时文件。
- 性能：录制时 UI 不应明显卡顿。
- 文件安全：取消录制时进入回收站或临时目录，不直接静默删除。
- 兼容性：优先支持 Windows。
- 可维护性：录制、素材、编排、剪映导出模块分层。
- 可观察性：保留录制和导出日志。
- 退出完整性：关闭软件后不应残留应用主进程、渲染进程或子进程。

- 保留配置 JSON。
- 对接素材编排脚本。
- 支持 `_events.json` 智能缩放数据。

## 13. MVP 验收清单

- 用户可以设置保存目录。
- 用户可以设置录制范围，并保存默认录制范围。
- 用户可以完成一次普通录制，并在保存目录看到 mp4。
- 用户可以切换到多段录制。
- 用户可以连续录制多段，所有片段显示在素材面板。
- 用户可以预览、删除、重命名、排序片段。
- 用户可以合并导出多段视频。
- 用户可以直接生成剪映草稿。
- 草稿生成时保留剪映草稿配置 JSON。
- 带事件记录的视频可以生成同名 `_events.json`。
- 用户关闭软件后，任务管理器中不残留本应用相关进程。

## 14. 已确认产品口径

- 简录制器聚焦录制、素材整理和剪映草稿生成，不提供内容创作画布能力。
- 录制源为整个屏幕或指定窗口。
- 录制范围支持默认设置和常用画幅预设：16:9、4:3、3:4、9:16、1:1、自定义。
- 音频可配置：系统声音、麦克风、系统声音 + 麦克风、静音。
- 多段编排不需要字幕能力。
- MVP 需要直接生成剪映草稿，配置 JSON 作为内部中间产物保留。
- 不需要摄像头画中画。
- 鼠标事件 MVP 默认只记录点击事件；鼠标移动轨迹暂不作为必选功能。
